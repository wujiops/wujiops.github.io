FFmpeg Static Auto-Builds

FFmpeg 静态构建

Static Windows (x86_64) and Linux (x86_64) Builds of ffmpeg master and latest release branch.

FFmpeg-Builds 提供了 Windows (x86_64) 和 Linux (x86_64) 最新发布的 FFmpeg master 分支的静态构建。

Windows builds are targetting Windows 7 and newer.

Windows 构建针对 Windows 7 及更高版本。

Linux (x86_64) builds are targetting Ubuntu 16.04 (glibc-2.23 + linux-4.4) and anything more recent.

Linux (x86_64) 构建针对 Ubuntu 16.04 (glibc-2.23 + linux-4.4) 及更新版本。

Linux (arm64) builds are targetting Ubuntu 18.04 (glibc-2.27 + linux-4.15) and anything more recent.

Linux (arm64) 构建针对 Ubuntu 18.04 (glibc-2.27 + linux-4.15) 及更新版本。

Auto-Builds

自动构建

Builds run daily at 12:00 UTC (or GitHubs idea of that time) and are automatically released on success.

每天中午 12:00 UTC运行一次自动构建，并在成功时自动发布。

Auto-Builds run ONLY for win64 and linux(arm)64. There are no win32/x86 auto-builds, though you can produce win32 builds yourself following the instructions below.

仅针对 win64 和 linux(arm64) 进行自动构建。没有 win32/x86 的自动构建，但您可以按照以下说明自行生成 win32 构建。

忽略自动构建部分，自己构建问题太多。

Release Retention Policy

本仓库构建发布保留策略

  • The last build of each month is kept for two years.

    每个月的最后一个构建将保留两年。

  • The last 14 daily builds are kept.

    保留最近的 14 天的构建。

  • The special “latest” build floats and provides consistent URLs always pointing to the latest build.

    “latest” 构建始终指向最新的构建。

The linuxarm64 target will not build some dependencies due to lack of arm64 (aarch64) architecture support or cross-compiling restrictions.

由于缺乏arm64 (aarch64)架构支持或交叉编译限制，linuxarm64目标将不构建某些依赖项。

  • davs2 and xavs2: aarch64 support is broken.
  • libmfx and libva: Library for Intel QSV, so there is no aarch64 support.

Available variants:

本仓库可用的构建：

没有什么好推荐的，各人按照各人的需要自行选择，我选择的是gpl版本

  • gpl Includes all dependencies, even those that require full GPL instead of just LGPL.

    gpl 包括所有依赖项，即使那些需要完整的GPL而不是仅LGPL的依赖项。

  • lgpl Lacking libraries that are GPL-only. Most prominently libx264 and libx265.

    lgpl 缺少仅供GPL使用的库。最显著的是libx264和libx265。

  • nonfree Includes fdk-aac in addition to all the dependencies of the gpl variant.

    nonfree 包括gpl变体的所有依赖项以及fdk-aac。

  • gpl-shared Same as gpl, but comes with the libav* family of shared libs instead of pure static executables.

    gpl-shared 与gpl相同，但附带libav*家族的共享库而不是纯静态可执行文件。

  • lgpl-shared Same again, but with the lgpl set of dependencies.

    lgpl-shared 再次相同，但附带lgpl集的依赖项。

  • nonfree-shared Same again, but with the nonfree set of dependencies.

    nonfree-shared 再次相同，但附带nonfree集的依赖项。

阿里云服务器

仅用于个人学习
阿里云抢占式实例每小时约2元，前提要充值100元余额。可以随时提现
当然也可以选择华为云、腾讯云等相应的产品
一个小时的时间和1G流量足够使用了，如果你是新手应该多在虚拟机上练习安装，多看些其它网站博客教程，避免财产损失。

其它教程有写将CUDA驱动lib编译进ffmpeg。实际测试我并没有这样做。
我只在GPU裸金属服务器安装上了CUDA驱动，以yum的方式安装的(也可以运行cuda_12.1.1_530.30.02_linux.run安装)。在FFmpeg-Builds仓库下载了gpl的编译包。直接解压到了我的/usr/local/ffmpeg6目录，配置了一下PATH，或者直接使用/usr/local/ffmpeg6/bin/ffmpeg [参数1] [参数2] … 直接就可以使用GPU加速转码。
