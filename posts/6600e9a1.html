安装hexo

安装前提：先安装nodejs和git，win10和Linux安装建议使用安装包，安装到默认位置即可，省的配置环境变量。

Node.js

Git

进入你的hexo目录，建议使用vscode或者typora打开目录

npm i hexo hexo-cli
hexo init
npm i
# 新建完成后，指定文件夹的目录如下：
.
├── node_modules
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

在win10系统，如果使用hexo s报错，配置win10的环境变量：

打开设置，搜索高级系统设置，找到环境变量，在用户变量和系统变量里面找到Path，分别点击编辑

新建一行，这一行是你的hexo目录下面的node_modules下的.bin目录

直接在文件资源管理器打开.bin目录，复制地址栏的地址即可。

安装主题

进入到themes目录，克隆主题，并修改文件夹名称为matery，删除如果你需要将主题自定义美化，并上传到git仓库，需要将clone下来的仓库中的.git目录删除，在hexo目录创建.git，上传到自己的代码库统一管理。

git clone https://github.com/blinkfox/hexo-theme-matery.git

在hexo目录将_config.landscape.yml修改为_config.matery.yml

统一配置文件

将主题文件夹下的_config.yml文件中的内容复制到hexo根目录下的_config.matery.yml文件中，并删除主题文件夹下的_config.yml文件

以后主题配置文件就是_config.matery.yml，hexo配置文件是_config.yml

写作技巧

写草稿，不会被发布：

hexo n draft 文章草稿

这样会在source/_draft中新建一个文章草稿.md，预览草稿：

hexo s -p 80 --draft

不预览草稿，正式发布预览：

hexo s -p 80

写完的草稿，正式发布：

hexo p draft 文章草稿

这样会自动把文章草稿.md发布到_posts目录下。

写作请使用

hexo n post --path 分类名/文章名 "title信息"

hexo简写命令：

hexo n "文章名" == hexo new "文章名"
hexo p == hexo publish
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy
hexo s   # Hexo会监视文件变动并自动更新，您无须重启服务器。
hexo s -s   # 静态模式
hexo s -p 5000   # 更改端口
hexo s -i 192.168.1.10   # 自定义 IP

hexo clean   # 清除缓存,网页正常情况下可以忽略此条命令
hexo g   # 生成静态网页
hexo d   # 开始部署
