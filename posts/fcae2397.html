Python项目的组织结构：

直接表现形式：

包 —— 文件夹

名字就是文件夹名（包里面可以有子包，文件夹中有文件夹，很灵活）

模块 —— *.py文件

名字就是文件名，区分不同包的相同名的模块

class —— 类
写在.py文件里

命名空间：（abc.c4和xyz.c4）abc和xyz是文件夹名（包名），c4是同名文件，不同包。abc.c4称为路径也是命名空间

包和文件夹区分：包里面包含一个__init__.py文件（可以是空白文件，可以是模块）引用__init__.py模块，直接引用包名即可

文件夹里没有__init__.py就是一个普通文件夹

-----------------------分割线-----------------------
import(导入模块使用麻烦)、from..import(导入模块直接使用，不建议直接使用import *)、import..as(对import导入设置别名)，避免循环导入

约定目录结构：

(py3) [root@bogon workdir]# tree A
A
├── A.py
├── B
│   └── B.py
└── C
    └── C.py

内容：

(py3) [root@bogon A]# cat B/B.py  #设置变量
a = 1
b = 2
(py3) [root@bogon A]# cat C/C.py  #设置变量
c = 3
d = 4
(py3) [root@bogon A]# cat A.py 
import B.B   #导入B/B.py模块
print (B.B.a)   #打印B.py模块里 的变量a
(py3) [root@bogon A]# python A.py 
1
(py3) [root@bogon A]# cat A.py 
from B.B import a   #导入B.py里面的a变量
print(a)   #直接使用
(py3) [root@bogon A]# python A.py 
1
(py3) [root@bogon A]# cat A.py 
import B.B as NAME   #设置别名
print (NAME.a,NAME.b)   #打印别名+变量名
(py3) [root@bogon A]# python A.py 
1 2
(py3) [root@bogon A]# cat A.py 
from B.B import a, \   #换行方式
    b
print(a, b)
(py3) [root@bogon A]# python A.py 
1 2

(py3) [root@bogon A]# cat A.py 
import B   #直接引用包B
(py3) [root@bogon A]# python A.py

包的引用问题

$ tree workdir 
workdir
├── README.md
├── test1
│   ├── c1.py
│   ├── __pycache__
│   │   └── c1.cpython-37.pyc
│   └── test2
│       ├── c1.py
│       └── __pycache__
│           └── c1.cpython-37.pyc
└── test2
    ├── c2.py
    └── __pycache__
        └── c2.cpython-37.pyc
A=/workdir/test1/c1.py
B=/workdir/test1/test2/c1.py
C=/workdir/test2/c2.py

A可以调用B

C可以调用A

A再调用C出错

注意:

包的引用，个人理解：被引用的模块中加入 if name == ‘main‘: 函数，这样可以只引用函数，不会执行 “main”。

被引用模块加入 all = [‘a’, ‘c’]，引用时使用 from model.mod import *，这样只会引入被引模块all定义的函数，不会引入其他函数

将函数定义在一个包中，这样比较好一点

相互引用会造成一些错误。当被引模块出现 sys.path.append(‘..’) 的时候，或者其他状况。
