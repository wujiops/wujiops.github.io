安装

# 初始环境配置
sysctl -w vm.max_map_count=524288
sysctl -w fs.file-max=131072
ulimit -n 131072
ulimit -u 8192

# 创建目录
mkdir /home/sonarqube/{postgresql,sonarqube/{extensions,data,logs}} -p
chmod 777 /home/sonarqube/sonarqube/*

# 拉取PG数据库并运行
docker run --restart=always --name pgdb -e POSTGRES_USER=sonar -e POSTGRES_PASSWORD=sonar -e POSTGRES_DB=sonar -p 5432:5432 -v /home/sonarqube/postgresql:/var/lib/postgresql -d postgres

# 安装最新的SonarQube
docker run --restart=always --name sonarqube --link pgdb -e SONAR_JDBC_USERNAME=sonar -e SONAR_JDBC_PASSWORD=sonar -e SONAR_JDBC_URL=jdbc:postgresql://pgdb:5432/sonar -p 19000:9000 -v /home/sonarqube/sonarqube/data:/opt/sonarqube/data -v /home/sonarqube/sonarqube/extensions:/opt/sonarqube/extensions -v /home/sonarqube/sonarqube/logs:/opt/sonarqube/logs -d sonarqube:lts-community

# 忘记使用 --restart=always 的话可以使用
# docker update --restart=always sonarqube
# docker update --restart=always pgdb
# 对运行的容器增加自动重启配置

部署插件

# 插件的位置/home/sonarqube/sonarqube/extensions/plugins
# 有时候插件市场更新缓慢，需要手动下载安装，需要重启
wget https://ghproxy.com/https://github.com/xuhuisheng/sonar-l10n-zh/releases/download/sonar-l10n-zh-plugin-10.1/sonar-l10n-zh-plugin-10.1.jar
cp sonar-l10n-zh-plugin-10.1/sonar-l10n-zh-plugin-10.1.jar /home/sonarqube/sonarqube/extensions/plugins/
# 别忘了对插件和目录授权
docker restart sonarqube

通用这里要配置SonarQube代理域名

认证这里选择gitlab，按照提示填写即可，注意这里的注册要打开，意思是gitlab用户登录SonarQube，自动注册SonarQube用户。不勾选的话需要手动在SonarQube用户添加gitlab用户。

下面勾选组同步，需要gitlab授权API，如果只登录，gitlab只授权readapi即可。

问题：Java Runtime (class file version 55.0), this version of the Java Runtime only recognizes class file versions up to 52.0

要使用JDK11，我是用dnf安装的jdk8和jdk11，目录为/usr/lib/jvm/默认我是JDK8

使用export JAVA_HOME=/usr/lib/jvm/java-11，再进行mvn clean verify...即可。
