docker registry 生成本地私有镜像库

获取 registry　镜像

docker pull registry

运行创建容器，/var/lib/registry挂载到本地

docker run -d -p 80:5000 --restart=always --name registry -v /opt/myregistry:/var/lib/registry registry

新增本地镜像仓库地址

cat > /etc/docker/daemon.json << EOF
{
  "registry-mirrors": ["https://fogd2pyo.mirror.aliyuncs.com"],
  "insecure-registries": ["docker.registry.com"]
}
EOF
IP=`ip a | grep -E "^    inet .* ens| eth.*$" | grep -v : | awk -F' ' '{print $2}' | awk -F/ '{print $1}'`
echo "$IP docker.registry.com" >> /etc/hosts
echo "其他机器请在hosts中映射 $IP docker.registry.com"

重启docker

systemctl daemon-reload
systemctl restart docker

获取nginx镜像

docker pull nginx

重新打上本地的仓库的tag

docker tag nginx docker.registry.com/nginx

将打好标签的镜像push到本地仓库

docker push docker.registry.com/nginx

验证

curl访问

curl http://docker.registry.com/v2/_catalog
# {"repositories":["nginx"]}   # 可以看到有nginx镜像

补充：

配置证书

创建证书，有效期设置10年

mkdir ~/.certs
openssl req -newkey rsa:4096 -nodes -sha256 -keyout ~/.certs/docker.registry.com.key -x509 -days 3650 -out ~/.certs/docker.registry.com.crt

带证书运行registry容器

docker run -d \
--restart=always \
--name registry \
-v ~/.certs/:/certs \
-e REGISTRY_HTTP_ADDR=0.0.0.0:443 \
-e REGISTRY_HTTP_TLS_CERTIFICATE=/certs/docker.registry.com.crt \
-e REGISTRY_HTTP_TLS_KEY=/certs/docker.registry.com.key \
-p 443:443 \
-v /opt/registry:/var/lib/registry \
registry:latest

测试

docker pull httpd   # 从官方pull httpd镜像
docker tag httpd docker.registry.com/httpd   # 重新打标签
docker push docker.registry.com/httpd   # 将打好标签的镜像push到本地仓库
curl https://docker.registry.com/v2/_catalog -k   # 验证
{"repositories":["docs","httpd","nginx"]}   # 可以看到本地镜像库中已有的镜像

注意：

向服务器发送 https 请求

curl需要加上-k，wget需要加上--no-check-certificate

其他主机访问仓库

cat /etc/docker/daemon.json   # 配置insecure-registries
{
  "registry-mirrors": ["https://fogd2pyo.mirror.aliyuncs.com"],
  "insecure-registries": ["docker.registry.com"]
}

192.168.1.40 docker.registry.com   # 主机映射

docker pull docker.registry.com/httpd   # pull镜像

基于用户密码访问

使用htpasswd工具，生成验证文件

yum -y install httpd-tools

htpasswd -Bbn roxn 123456 > ~/.certs/.htpasswd   # 这里我和ssl加密文件放在一个文件夹下

带证书和用户认证运行registry容器

docker run -d \
--restart=always \
--name registry \
-v ~/.certs/:/certs \
-e REGISTRY_HTTP_ADDR=0.0.0.0:443 \
-e REGISTRY_HTTP_TLS_CERTIFICATE=/certs/docker.registry.com.crt \
-e REGISTRY_HTTP_TLS_KEY=/certs/docker.registry.com.key \
-e REGISTRY_AUTH=htpasswd \
-e REGISTRY_AUTH_HTPASSWD_REALM="Registry Realm" \
-e REGISTRY_AUTH_HTPASSWD_PATH=/certs/.htpasswd \
-p 443:443 \
-v /opt/registry:/var/lib/registry \
docker.registry.com/registry:latest

docker-compose.yml

执行docker-compose up -d效果和上面的命令相同

registry:
  restart: always
  image: docker.registry.com/registry
  container_name: registry
  ports:
    - 443:443
  environment:
    - REGISTRY_AUTH=htpasswd
    - REGISTRY_AUTH_HTPASSWD_REALM=Registry_Realm
    - REGISTRY_AUTH_HTPASSWD_PATH=~/.certs/.htpasswd
    - REGISTRY_HTTP_ADDR=0.0.0.0:443
    - REGISTRY_HTTP_TLS_CERTIFICATE=/certs/docker.registry.com.crt
    - REGISTRY_HTTP_TLS_KEY=/certs/docker.registry.com.key
  volumes:
    - ~/.certs/:/auth
    - /opt/registry:/var/lib/registry

查看本地仓库的镜像

curl https://docker.registry.com/v2/_catalog -k
{"errors":[{"code":"UNAUTHORIZED","message":"authentication required","detail":[{"Type":"registry","Class":"","Name":"catalog","Action":"*"}]}]}

此种方法与上述使用证书的区别在于，此时查看文件_catalog中的内容时，发现是加密过的，无法显示

注意：

  1. _catalog相当于一个API接口,查看私有仓库中有哪些镜像

  2. 必须先做加密后做认证，因为如果反着来就没有意义，直接可以通过数据包知道密码
