系统配额

  1. 配额的作用：当Linux根分区的磁盘空间耗尽时，Linux系统将无法再建立新的文件（包括程序运行的临时文件），从而出现服务程序崩溃，系统无法启动等故障现象。

为了避免在服务器中出现类似的磁盘空间不足的问题，可以设置启用磁盘配额功能，对用户在指定文件系统（分区）中使用的磁盘空间、文件数量进行限制，以防个别用户恶意或无意间占用大量磁盘空间。是为了存储空间的稳定性和可持续性。

  1. 配额系统综述：

在内核中执行

作用范围：以文件系统（分区）为单位启用

限制对象：对不同组、用户限制

限制类型：根据磁盘容量或者文件数量（配置文件中容量单位默认为K）

限制方法：

软限制（soft）：指定一个软性的配额数值，在固定的宽限期内（默认7天），允许超过这个限制，但文件系统会给出警告信息。

硬限制（hard）：指定一个硬性的配额数值，绝对禁止用户超过的限制值，当达到硬限制值时系统也会给出警告并禁止继续写入数据，硬限制的配额值应大于相应的软限制值，否则软限制将失效。

综上，只有当用户或组、文件系统（分区）及配额数值都满足限额条件时，才会对操作进行限制。

  1. 配额管理：步骤：以支持配额功能的方式挂载文件系统→检测磁盘配额并生成配额文件→编辑用户和组行号的配额设置→开启文件系统配额功能→验证磁盘配额功能→查看配额使用情况。

以支持配额功能的方式挂载

mount -o usrquota,grpquota 设备分区名 挂载点

以支持配额功能的方式重新挂载

mount -o remount,usrquota,grpquota 设备分区名 挂载点

开机自动挂载支持磁盘配额功能

vim /etc/fstab

设备分区名 挂载点 文件系统类型 usrquota,grpquota 0 0

检测磁盘配额并生成配额文件

quotacheck -augcv

选项

-a：检测所有分区（如不使用-a，则必须指定一个设备分区名或挂载点，作为命令参数）

-u：检测用户配额信息

-g：检测组配额信息

-c：创建新的配额文件

-v：显示命令执行过程

设定配额：编辑用户和组的配额设置

edquota -u 用户名

（blocks设置soft 5文件大小软限制为5k，设置hard 9文件大小硬限制为9k；inodes设置soft 2文件数量软限制为2个，设置hard 5文件数量硬限制为5个）

edquota -g 组名

打开配额文件编辑

block soft hard inode soft hard
        1   2           1   2

释义：文件大小 软限制1K 硬限制2K 文件个数 软限制1个 硬限制2个

修改宽限期

edquota -t

修改配置文件 #days

启动配额功能

quotaon -u 设备分区名（挂载点）

quotaon -g 设备分区名（挂载点）

-v 显示命令的执行过程

关闭磁盘配额

quotaoff -u/g -v 设备分区名（挂载点）

查看配额状态

quota -u 用户名

quota -g 组名

查看指定文件系统输出配额使用情况

repquota 设备分区名

查看所有可用分区的配额使用报告

repquota -a

软RAID设备

RAID：磁盘阵列

简单的说，RAID是把多块独立的物理硬盘组合起来，形成一个硬盘组即逻辑硬盘，从而提供比单个硬盘更高的存储性能和提供数据备份技术。

组成磁盘阵列的不同方式称为RAID级别，根据不同级别，分别可以提供不同的速度、安全和性价比，根据实际情况选择适当的RAID级别可以满足用户对存储系统可用性能和容量的要求。

RAID实现方式：

外接式磁盘阵列：通过扩展卡提供适配能力

内接式RAID：主板集成RAID控制器；安装OS前在BIOS里配置

软件RAID：通过OS实现

RAID级别：0：条带卷 1：镜像卷 5、6、1 0、0 1、

RAID-0：条带卷

数据分割之后并行存储

读写性能提升

可用空间N x min（N磁盘数量，min最小磁盘容量）

无容错能力（损坏一块就不能用），适合对数据读写要求高，而对安全无要求的环境

至少磁盘数：2

RAID-1：镜像卷

数据分割后，进行镜像备份存储

读取性能提升，写入性能下降

可用空间min

有冗余能力（有一块磁盘完好就行），适合对安全要求高的环境

至少磁盘数：2或2的倍数

RAID-5：

3块以上的盘组成阵列，一个数据分成N-1份，同时生成一份校验数据，共N份数据在N块盘上循环均衡存储

校验数据作用：在某一块盘损坏时，可以通过其它盘和校验数据，把数据内容算出来

N块盘同时读写，由于校验机制，写性能相对不高，但综合性能很好

可用空间N-1 x min

有容错能力：允许坏一块，不影响数据

最少磁盘数：3

RAID-6：（类似5）

由4块以上的盘组成阵列，数据分成N-2份，并生成2份独立的奇偶校验数据，N份数据在磁盘上循环均衡存储

由于存在2个独立的奇偶校验数据，所以写入性能较差，但数据可靠性非常高

有容错能力：最多允许坏2块盘

最少磁盘数：4

RAID-10：先进行RAID-1再进行RAID-0

RAID-01：先进行RAID-0再进行RAID-1

其它RAID-50等

步骤：安装mdadm→准备用于RAID阵列的分区→创建RAID设备→在RAID设备中建立文件系统→挂载并使用文件系统

fdisk /dev/sd*

mdadm [模式] <设备名称> [选项] <磁盘分区>

模式：

-C：创建

    -n #：使用#块设备创建此RAID

    -l #：指明要创建的RAID级别

    -a yes：自动创建目标RAID设备文件

    -c #[KMG]：指明块大小

    -x #：指明空闲盘的个数

-A：装配、激活

    -s：扫描设备信息

-D（--detail）：显示RAID的详细信息

mdadm -D /dev/md#

管理模式：

    -f：标记指定磁盘为损坏

    -a：添加磁盘

    -r：移除磁盘

观察md的状态：

cat /proc/mdstat

创建RAID设备：

mdadm -C /dev/md0 -a yes -l 5 -n 3 -x 1 /dev/sdb{1,2,3,4}

在RAID中建立文件系统：

mkfs.ext4 /dev/md0

挂载使用

mount /dev/md0 /testdir

生成RAID配置文件：/etc/mdadm.conf（默认不存在）

执行mdadm -D -s >> /etc/mdadm.conf

vim /etc/mdadm.conf

修改其内容：在最上面添加一行：DEVICE /dev/sd* ；第二行删除metadata=…,name=…两项；保存退出

检测配置文件是否有语法错误：

使用启动/停止服务的方式：

  停服务：mdadm -S /dev/md0

  激活/启动：mdadm -As /dev/md0

  强制启动：mdadm -R /dev/md0

查看df -hT

自动挂载：添加到/etc/fstab文件中去

模拟磁盘故障

mdadm /dev/md0 -f /dev/sdb1

移除磁盘

mdadm /dev/md0 -r /dev/sdb1

添加新成员

mdadm -G /dev/md0 -n 4 -a /dev/sdb1

从软件RAID磁盘修复磁盘故障：

替换出故障的磁盘然后开机；在备用驱动器上重建分区；

mdadm /dev/md0 -a /dev/sdb1

查看状态：

mdadm --query或cat /proc/mdstat

删除RAID信息：mdadm --zero-superblock /dev/sdb1

LVM逻辑卷

允许对卷进行方便操作的抽象层，包括重新设定文件系统的大小，允许在多个物理设备间重新组织文件系统

将设备指定为物理卷PV

用一个或多个物理卷来创建一个卷组VG

物理卷是用固定大小的物理区域PE来定义的

在物理卷上利用物理区域创建逻辑卷LV

可以在逻辑卷上创建文件系统

LVM介绍：LVM：逻辑卷管理器；设备名：/dev/dm-#

LVM可以弹性的更改LVM的容量：通过交换PE来进行资料的转换，将原来LV内的PE转移到其他的设备中以LV的容量，或将其他设备中的PE加到LV中以加大容量

功能 物理卷管理PV 卷管理VG 逻辑卷管理LV
scan 扫描 pvscan vgscan lvscan
create 建立 pvcreate vgcreate lvcreate
display 显示 pvdisplay vgdisplay lvdisplay
remove 删除 pvremove vgremove lvremove
reduce 缩减 - vgreduce lvreduce
extend 扩展 - vgextend lvestend

PV管理工具：

PV：物理卷：整个硬盘，或使用fdisk工具建立的普通分区（8e），包括许多默认的4MB大小的PE单元

创建PV物理卷

pvcreate /dev/sdb{1,2,3}

pvs：显示简要pv信息（pvdisplay显示的更详细）

VG管理工具

VG：卷组：一个或多个物理卷组合而成的整体（相当于逻辑上的一个硬盘）

创建卷组（-s指定PE大小8M）

vgcreate -s 8m vg名 pv名

vgs：显示简要vg信息（vgdisplay显示的更详细）

vgextend vg名 pv名（扩容vg）

vgreduce vg名 pv名（减容vg）

vgremove /dev/vg名（删除vg），先做pvmove /dev/pv1 /dev/pv2（将pv1中的内容移动到pv2中），不需要保留数据则不做pvmove

LV管理工具

LV：逻辑卷：从卷组中割出的一块空间，可以格式化挂载使用

创建逻辑卷

lvcreate -L 5G -n lv名 vg名（-L指定lv大小5g）

lvs：显示lv简要信息（lvdisplay显示的更详细）

扩展逻辑卷LV

lvextend -L 7G /dev/vg名/lv名（扩展逻辑卷到7G），

执行resize2fs /dev/vg名/lv名，让文件系统df -h识别扩展后的逻辑卷大小

缩减逻辑卷

umount /dev/vg名/lv名：卸载

e2fsck -f /dev/vg名/lv名：磁盘检查

resize2fs /dev/vg名/lv名 3G：指定缩减后的逻辑卷大小为3G（将分散的数据整理到要保留的3G逻辑卷空间里）

lvreduce -L 3G /dev/vg名/lv名：缩减逻辑卷为3G

mount /dev/vg名/lv名 /原挂载点（由于执行resize2fs命令，所以原挂载的内容会被保留，原挂载点数据<3G）
