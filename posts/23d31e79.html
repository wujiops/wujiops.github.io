Vmware磁盘扩容

vmware disk parted lvm
Notes
发布日期:   2023-04-14
更新日期:   2023-08-15
文章字数:   851
阅读次数:  

更正：虚拟机安装的时候磁盘分区是默认未配置，或者选择lvm逻辑卷管理才可使用本方法，遇到BUG和ERR欢迎留言或+Q

前言：编写lamp编译安装文档的时候，MySQL8.0 cmake完成，make编译的时候磁盘空间不足（原/有17G，一共20G，swap2G，boot1G）

故有此次扩容文档，适用于服务器系统盘扩容（不建议这样做，应当提前规划好各系统目录的作用，做好逻辑卷等等）

不涉及到关机操作，应急使用，操作步骤：

  1. 打开要扩容的虚拟机设置–>选中磁盘–>点击扩展，输入扩容后的大小，等待扩容完成。

VMware 磁盘扩容

进入虚拟机经行命令操作

命令下是执行此命令打印的信息，删除了大部分，留下的是关键信息部分

查看可用磁盘大小，我的是20G扩容到40G

fdisk -l
# Disk /dev/nvme0n1：40 GiB，42949672960 字节，83886080 个扇区

仍然显示17G

df -h
# 文件系统             容量  已用  可用 已用% 挂载点
# /dev/mapper/cl-root   17G  1.8G   16G   11% /

对其进行新的分区

由fdisk -l得到的信息，对其进行新的分区，除了第一次输入n，最后一次输入w，其它可以直接回车

fdisk /dev/nvme0n1
# 命令(输入 m 获取帮助)：n
# 选择 (默认 p)：
# 分区号 (3,4, 默认  3): 
# 第一个扇区 (41943040-83886079, 默认 41943040): 
# 上个扇区，+sectors 或 +size{K,M,G,T,P} (41943040-83886079, 默认 83886079): 
# 创建了一个新分区 3，类型为“Linux”，大小为 20 GiB。
# 命令(输入 m 获取帮助)：w

查看新建的分区

lsblk -l
# nvme0n1p3 259:3    0  20G  0 part

对新分区做物理卷

pvcreate /dev/nvme0n1p3
# Physical volume "/dev/nvme0n1p3" successfully created.

查询root分区所在的卷组

vgdisplay -v
# --- Logical volume ---
# LV Path                /dev/cl/root
# LV Name                root   # 逻辑卷名
# VG Name                cl   # 卷组名

扩展cl卷组

扩展cl卷组，即把物理卷/dev/nvme0n1p3分配到cl卷组

vgextend cl /dev/nvme0n1p3
# Volume group "cl" successfully extended

将cl卷组的剩余容量全部分给根分区

lvextend -l +100%FREE /dev/cl/root
# Size of logical volume cl/root changed from <17.00 GiB (4351 extents) to 36.99 GiB (9470 extents).
# Logical volume cl/root successfully resized.

同步文件系统

同步文件系统，另外一种写法xfs_growfs /dev/cl/root报错，酌情使用。

xfs_growfs  /
# meta-data=/dev/mapper/cl-root    isize=512    agcount=4, agsize=1113856 blks
#          =                       sectsz=512   attr=2, projid32bit=1
#          =                       crc=1        finobt=1, sparse=1, rmapbt=0
#          =                       reflink=1
# data     =                       bsize=4096   blocks=4455424, imaxpct=25
#          =                       sunit=0      swidth=0 blks
# naming   =version 2              bsize=4096   ascii-ci=0, ftype=1
# log      =internal log           bsize=4096   blocks=2560, version=2
#          =                       sectsz=512   sunit=0 blks, lazy-count=1
# realtime =none                   extsz=4096   blocks=0, rtextents=0
# data blocks changed from 4455424 to 9697280

查看扩容后的根分区大小

df -h
# /dev/mapper/cl-root   37G  2.0G   36G    6% /

大磁盘扩容

使用工具分区

查看新增加的磁盘信息

fdisk -l


新建分区（fdisk最大支持2T磁盘）

parted /dev/vdd

mklabel gpt
mkpart primary 0% 100%
quit


再次查看磁盘信息

fdisk -l

LVM扩容

创建PV

pvcreate /dev/vdd1


查看VG信息

df -h

vgdisplay


VG扩容

vgextend volume-nfs /dev/vdd1


使用Free空间

lvextend -l +100%FREE /dev/mapper/volume--nfs-lv--nfs


刷新VG信息

xfs_growfs /dev/mapper/volume--nfs-lv--nfs
# 报错：xfs_growfs: /dev/mapper/xxx is not a mounted XFS filesystem 使用下面的命令。
xfs_growfs /

验证磁盘容量

刷新磁盘挂载信息

resize2fs -p -F /dev/mapper/volume--nfs-lv--nfs


查看磁盘挂载信息

df -h
