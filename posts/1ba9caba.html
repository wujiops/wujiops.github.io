版本选择

rabbitmq和erlang版本对应关系

这里选择了rabbitmq3.8.22和erlang24.0.5

大部分不能启动或者其它报错都是版本不合引起的

安装rabbitmq

在hosts文件中写上3台机器的IP和主机名对应关系

192.168.1.1 rabbitmq-01
192.168.1.2 rabbitmq-02
192.168.1.3 rabbitmq-03

修改系统配置：vi /etc/sysctl.conf

fs.file-max = 100000

使设置生效：sysctl -p

查看系统限制：sysctl fs.file-max

调整用户限制：vi /etc/security/limits.conf

*              soft     nofile          65536
*              hard     nofile          65536

重启系统使之生效，检查用户限制是否生效：ulimit -n

详细ulimit配置文件修改参阅

RabbitMQ相关端口

4369 (epmd), 25672 (Erlang distribution) 
5672, 5671 (AMQP 0-9-1 without and with TLS) 
15672 (if management plugin is enabled) 
61613, 61614 (if STOMP is enabled) 
1883, 8883 (if MQTT is enabled)

三台机器分别执行：

# 添加EPEL源
rpm -Uvh https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm
# 添加Erlang源
yum install -y wget
rpm -Uvh http://packages.erlang-solutions.com/erlang-solutions-1.0-1.noarch.rpm
# 安装Java
yum makecache
yum install -y java java-devel
# 安装RabbitMQ
wget https://github.91chifun.workers.dev/https://github.com//rabbitmq/rabbitmq-server/releases/download/v3.8.22/rabbitmq-server-3.8.22-1.el7.noarch.rpm
rpm --import https://www.rabbitmq.com/rabbitmq-signing-key-public.asc
yum install -y rabbitmq-server-3.8.22-1.el7.noarch.rpm

创建目录、用户

mkdir -p /home/rabbitmq/{mnesia,log}
mkdir -pv /etc/rabbitmq
useradd rabbitmq
chown -R rabbitmq. /etc/rabbitmq/
chown -R rabbitmq. /home/rabbitmq/

修改数据目录

$ vi /etc/rabbitmq/rabbitmq-env.conf 
RABBITMQ_MNESIA_BASE=/home/rabbitmq/mnesia
RABBITMQ_LOG_BASE=/home/rabbitmq/log

设置每个节点Cookie

Rabbitmq的集群是依赖于erlang的集群来工作的，所以必须先构建起erlang的集群环境。

Erlang的集群中各节点是通过一个magic cookie来实现的，这个cookie存放在/var/lib/rabbitmq/.erlang.cookie中，文件是400的权限。

所以必须保证各节点cookie保持一致，否则节点之间就无法通信

chmod 700 /var/lib/rabbitmq/.erlang.cookie
echo -n "CGOEQMSYVLWHJPTBDRN" > /var/lib/rabbitmq/.erlang.cookie
chmod 400 /var/lib/rabbitmq/.erlang.cookie

建议在RabbitMQ服务启动前修改过cookie，如果RabbitMQ服务已经启动，修改cookie值后，必须重启RabbitMQ服务，这步很关键

systemctl restart rabbitmq-server

启动、查看运行状态

systemctl daemon-reload
systemctl start rabbitmq-server
systemctl status rabbitmq-server
systemctl enable rabbitmq-server

集群搭建

将 rabbitmq-01、rabbitmq-02、rabbitmq-03 组成集群：

默认是磁盘节点，如果是内存节点的话，需要加--ram参数

在rabbitmq-02、rabbitmq-03上分别运行：

rabbitmqctl stop_app
rabbitmqctl join_cluster rabbit@rabbitmq-01
rabbitmqctl start_app

集群用户、插件

在rabbitmq-01设置镜像策略、安装插件

rabbitmqctl set_policy ha-all "^" '{"ha-mode":"all","ha-sync-mode":"automatic"}'
rabbitmq-plugins enable rabbitmq_management

账号设置

rabbitmqctl set_permissions -p / admin ".*" ".*" ".*"
rabbitmqctl add_user rabbitmq Q4prKB6aGhqk

一些命令

# 添加用户；
add_user <username> <password>

# 删除一个用户；
delete_user <username>

# 改变用户密码（也是改变web管理登陆密码）；
change_password <username> <newpassword>

# 清除用户的密码，该用户将不能使用密码登陆，但是可以通过SASL登陆如果配置了SASL认证；
clear_password <username>

# 设置用户tags，管理员、普通用户等；
set_user_tags <username> <tag> ...

# 列出用户；
list_users

# 创建一个vhosts；
add_vhost <vhostpath>

# 删除一个vhosts；
delete_vhost <vhostpath>

# 列出vhosts；
list_vhosts [<vhostinfoitem> ...]

# 针对一个vhosts给用户赋予相关权限；
set_permissions [-p <vhostpath>] <user> <conf> <write> <read>

# 清除一个用户对vhosts的权限；
clear_permissions [-p <vhostpath>] <username>

# 列出哪些用户可以访问该vhosts；
list_permissions [-p <vhostpath>]

# 列出该用户的访问权限；
list_user_permissions <username>

# 其它
set_parameter （设置一个虚拟主机的运行时参数）
list_parameters (列出一个虚拟主机上的所有运行时参数)
clear_parameter (清除一个虚拟主机主机上的一个运行时参数)
set_global_parameter （设置一个全局的运行时参数）
list_global_paramters (列出全局的运行时参数)
clear_global_paramter (清除一个全局的运行时参数)

rabbitmq-env.conf中的每项都以RABBITMQ_为前缀，常用参数如下：

RABBITMQ_NODENAME=FZTEC-240088 //节点名称
RABBITMQ_NODE_IP_ADDRESS=127.0.0.1 //IP地址，空串bind所有地址，指定地址bind指定网络接口
RABBITMQ_NODE_PORT=5672 //TCP端口号，默认是5672
RABBITMQ_LOG_BASE=/data/rabbitmq/log //日志所在路径
RABBITMQ_PLUGINS_DIR=/data/rabbitmq/plugins //插件所在路径
RABBITMQ_MNESIA_BASE=/data/rabbitmq/mnesia //mnesia所在路径

第二章YUM安装rabbitmq

centos7 rabbitmq已停止维护：地址

# 默认epel库安装rabbitmq-server-3.3.5 如果够用可以安装epel源后安装rabbitmq-server
[root@centos7 ~]# yum install epel-release -y
# centos7安装高版本
[root@centos7 ~]# https://github.com/rabbitmq/erlang-rpm/releases/download/v23.3.4.18/erlang-23.3.4.18-1.el7.x86_64.rpm

# centos8+ 可以使用这种方法安装
[root@centos7 ~]# curl -1sLf 'https://dl.cloudsmith.io/public/rabbitmq/rabbitmq-erlang/setup.rpm.sh' | bash
[root@centos7 ~]# curl -s https://packagecloud.io/install/repositories/rabbitmq/rabbitmq-server/script.rpm.sh | bash
[root@centos7 ~]# yum -y install erlang rabbitmq-server
